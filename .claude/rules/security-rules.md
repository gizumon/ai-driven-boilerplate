# セキュリティルール

## 秘密情報管理

- APIキー、DBパスワード等の秘密情報はコードに含めない
- 環境変数または Secret Manager 経由で注入する
- `.env` ファイルは `.gitignore` に含める

## フロントエンド

- FCMサーバーキーをフロントエンドに配置しない
- API Base URL は環境変数で管理する

## バックエンド

- SQL インジェクション防止: sqlc のパラメータバインディングを使用
- 入力値バリデーションは Handler 層で実施
- エラーレスポンスに内部情報を含めない

## インフラ

- Terraform state に秘密情報を含めない
- IAM は最小権限原則に従う
